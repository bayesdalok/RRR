# Rural Risk Radar

3-month / 6-month rolling net cash flow forecasting, stress assessment, and
field-investigator/sector risk roll-up for 398 Gujarat microenterprises.

## Directory structure

```
rural-risk-radar/
├── data/
│   ├── raw/          # source files, one row per (entity, month) or per entity — never edited in place
│   ├── processed/     # joined panel, engineered features, train/val/test splits — all reproducible from raw/
│   └── external/      # anything not generated by this pipeline (e.g. transactions.csv, if ever supplied)
├── scripts/           # numbered, run in order — each script's output is the next script's only input
├── models/            # saved model artifacts (one subfolder per script that trains something)
├── reports/
│   ├── figures/        # plots
│   └── metrics/         # metrics as JSON, one file per evaluation run, timestamped
├── docs/
│   ├── modeling_roadmap.md   # the design doc: targets, leakage rules, feature/model plan
│   └── PROGRESS_LOG.md         # running log of what's done, what broke, what's still open — updated after every script
└── README.md
```

## Ground rules carried over from the roadmap

1. `enterprise_ground_truth.csv` is opened by exactly one script
   (`10_backtest_against_ground_truth.py`) and never merged into anything
   upstream of it.
2. No column derived from the *same-month* `income`/`expenses`/`net_cash_flow`
   is ever used contemporaneously as a feature — lag ≥ 1 only. See
   `docs/modeling_roadmap.md` §0 for the full table.
3. Every script is runnable standalone given only the previous script's
   output in `data/processed/` — no hidden state, no notebook-only steps.
